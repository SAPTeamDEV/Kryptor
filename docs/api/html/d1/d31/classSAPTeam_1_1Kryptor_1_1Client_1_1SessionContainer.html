◆ StartQueuedSessions()

void SAPTeam.Kryptor.Client.SessionContainer.StartQueuedSessions ( )

Starts registered sessions in a managed way.

This is the core of the Session based Application Model logic. This task scheduler logic works with checking the number of running sessions (sessions with status SessionStatus.Running) and checking the waiting sessions (sessions with status SessionStatus.NotStarted and be ready). The ready to run sessions determined by calling the ISession.IsReady(CancellationToken) method, if the return value is true the session will be added to the waiting list, otherwise it will be ignored and rechecked at the next run of the task scheduler. There is some important notes about this process.

The task scheduler only runs after each NewSession(ISession, bool, bool)calls and session ends. so if there is two sessions and the second session does not get ready until first session end, the task scheduler won't try to run that session and it will stuck at waiting status.

The ISession.IsReady(CancellationToken) should not throw ANY exceptions at all. it may break the scheduler. Although there is some layers to handle exceptions, but the ISession.IsReady(CancellationToken) is only allwed to response with true or false.

◆ WaitAll()

async Task SAPTeam.Kryptor.Client.SessionContainer.WaitAll ( CancellationToken cancellationToken)

Waits until all sessions have been ended.

This method also waits for sessions that are not started yet. If the cancellationToken was cancelled, this method will cancel all existing tokens in this container and waits for all of them to end. This method intended to be used by sub-session models. in standard session host based model, the monitoring, cancelling and ending is managed by various parts. But in sub-session models, you could achieve all of these features in this method. Using this method in session host is not recommended at all because this method is not intended for that porpose.

Parameters
cancellationTokenThe token to monitor for cancellation requests.

◆ WaitForRunningTasks()

void SAPTeam.Kryptor.Client.SessionContainer.WaitForRunningTasks ( )

Waits for all running tasks at the moment to end.

This method just waits for running tasks and does not care about waiting sessions (as they don't have a registered task). A Task must be registered by AddMonitoringTask(Task) that waits for not started sessions before calling this method. This method only intended to be used in ISessionHost.End(bool). so using it in sub-session models is not recommended as it could be lead to unexpected session exit and losing control of sub-sessions. it's highly recommended to use WaitAll(CancellationToken) method in that situation.
