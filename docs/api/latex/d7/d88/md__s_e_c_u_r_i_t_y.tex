The maintainers of this project take security seriously, and the reporting of potential security issues. If you believe you have found a security vulnerability in code from this repository that meets \href{https://docs.microsoft.com/previous-versions/tn-archive/cc751383(v=technet.10)}{\texttt{ Microsoft\textquotesingle{}s definition of a security vulnerability}}, please report it to us as described below.

We are using Microsoft\textquotesingle{}s vulnerability definition as it is public and familiar to .NET users.\hypertarget{md__s_e_c_u_r_i_t_y_autotoc_md10}{}\doxysection{\texorpdfstring{Reporting Security Issues}{Reporting Security Issues}}\label{md__s_e_c_u_r_i_t_y_autotoc_md10}
{\bfseries{Please do not report security vulnerabilities through public Git\+Hub issues.}}

Instead, please report them to the project maintainers using \href{https://keybase.io/aeliux}{\texttt{ keybase}} or \href{mailto:itsaeliux@gmail.com?subject=Kryptor\%20project\%20vulnerability}{\texttt{ email}}. You will typically receive a response within 24 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include the requested information listed below (as much as you can provide) to help us better understand the nature and scope of the possible issue\+:


\begin{DoxyItemize}
\item Type of issue (e.\+g. buffer overflow, SQL injection, cross-\/site scripting, etc.)
\item Full paths of source file(s) related to the manifestation of the issue
\item The location of the affected source code (tag/branch/commit or direct URL)
\item Any special configuration required to reproduce the issue
\item Step-\/by-\/step instructions to reproduce the issue
\item Proof-\/of-\/concept or exploit code (if possible)
\item Impact of the issue, including how an attacker might exploit the issue
\end{DoxyItemize}

This information will help us triage your report more quickly.\hypertarget{md__s_e_c_u_r_i_t_y_autotoc_md11}{}\doxysection{\texorpdfstring{Preferred Languages}{Preferred Languages}}\label{md__s_e_c_u_r_i_t_y_autotoc_md11}
