void SAPTeam.\+Kryptor.\+Client.\+Session\+Container.\+Start\+Queued\+Sessions (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}



Starts registered sessions in a managed way. 

This is the core of the \doxylink{class_s_a_p_team_1_1_kryptor_1_1_client_1_1_session}{Session} based Application Model logic. This task scheduler logic works with checking the number of running sessions (sessions with status \doxylink{namespace_s_a_p_team_1_1_kryptor_1_1_client_acfaef7f387541bf99fb0b3db2e77962aa5bda814c4aedb126839228f1a3d92f09}{Session\+Status.\+Running}) and checking the waiting sessions (sessions with status \doxylink{namespace_s_a_p_team_1_1_kryptor_1_1_client_acfaef7f387541bf99fb0b3db2e77962aafa7be7845bc42b3491d9d0377958be94}{Session\+Status.\+Not\+Started} and be ready). The ready to run sessions determined by calling the \doxylink{interface_s_a_p_team_1_1_kryptor_1_1_client_1_1_i_session_afc1437184579a5819ef206f6d16d0948}{ISession.\+Is\+Ready(\+Cancellation\+Token)} method, if the return value is {\ttfamily true} the session will be added to the waiting list, otherwise it will be ignored and rechecked at the next run of the task scheduler. There is some important notes about this process. 

The task scheduler only runs after each \doxylink{class_s_a_p_team_1_1_kryptor_1_1_client_1_1_session_container_a77cb6c6290c11cbb93dcbf55338baa62}{New\+Session(\+ISession, bool, bool)}calls and session ends. so if there is two sessions and the second session does not get ready until first session end, the task scheduler won\textquotesingle{}t try to run that session and it will stuck at waiting status. 

The \doxylink{interface_s_a_p_team_1_1_kryptor_1_1_client_1_1_i_session_afc1437184579a5819ef206f6d16d0948}{ISession.\+Is\+Ready(\+Cancellation\+Token)} should not throw ANY exceptions at all. it may break the scheduler. Although there is some layers to handle exceptions, but the \doxylink{interface_s_a_p_team_1_1_kryptor_1_1_client_1_1_i_session_afc1437184579a5819ef206f6d16d0948}{ISession.\+Is\+Ready(\+Cancellation\+Token)} is only allwed to response with {\ttfamily true} or {\ttfamily false}.
